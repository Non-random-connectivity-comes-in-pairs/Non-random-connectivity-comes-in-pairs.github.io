The full environment used for all computations referenced in the article is provided in form of a Docker image. In short, Docker uses virtualization on the operating system level to provide "containers" for Linux applications. To be able to access all information on this research project, please install Docker following the instructions given here.

Docker Image

The Docker image used for this project is named felix11h/aniso_netw_env and is available on Docker Hub. A short list of the most important dependencies included in this image are

  • Python 2.7.9
  • SciPy 0.14.0
  • NumPy 1.8.2
  • graphtool 2.12
  • Tex Live 2015
  • Sumatra 0.7

Accessing the Docker Container

The computational environment can be most easily accessed using the provided bash script access_lab.sh. In the directory,

source access_lab.sh

to start the container. Assuming you've installed Docker, the image felix11h/aniso_netw_env will be downloaded automatically if it's not available on your machine already (this will take some time and around 5 GB of disk space!).

In your terminal you should now see something along the lines of

docker@c30441c7f888:/home/lab$

indicating that you've successfully started the container and have now access to all of the tools listed above. You can now access the lab notebook through the Sumatra web interface. If you need more details on the Docker image and accessing it, see the troubleshooting section.
